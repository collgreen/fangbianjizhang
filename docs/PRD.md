# 产品需求文档 (PRD)

## 1. 产品概述

- **产品名称**：方便记账
- **产品定位**：Android 本地个人记账应用
- **目标用户**：有多账户管理需求的个人理财用户
- **核心价值**：简单易用、纯本地存储、数据安全可控、多账户全覆盖

## 2. 功能模块总览

```
┌─────────────────────────────────────────┐
│              方便记账 App                │
├──────────┬──────────┬─────────┬─────────┤
│   首页   │   资产   │  统计   │  设置   │
│  (Tab 1) │ (Tab 2)  │ (Tab 3) │ (Tab 4) │
```

| 优先级 | 功能 |
|--------|------|

## 4. 资产模块 (P0)

### 4.1 总资产概览

页面顶部显示：
- **净资产** = 资金账户总余额 + 投资账户总市值 - 信用账户已用总额 - 贷款剩余总额
- **总资产**（资金 + 投资）和 **总负债**（信用已用 + 贷款剩余）分别显示

### 4.2 资金账户

| 子类型 | 说明 |
|--------|------|
| 银行卡 | 储蓄卡、工资卡等 |
| 微信钱包 | 微信零钱 |
| 支付宝钱包 | 支付宝余额 |

字段定义：
- `name`: 账户名称（如"招商银行储蓄卡"）
- `balance`: 当前余额（Long，分）
- `icon`: 图标标识
- `include_in_total`: 是否计入总资产

余额变动规则：
- 支出 → balance 减少
- 收入 → balance 增加
- 转出 → balance 减少
- 转入 → balance 增加

### 4.3 信用账户

| 子类型 | 说明 |
|--------|------|
| 信用卡 | 各银行信用卡 |
| 花呗 | 支付宝花呗 |
| 白条 | 京东白条 |

字段定义：
- `name`: 账户名称
- `total_limit`: 总额度（Long，分）— 用户设置
- `used_amount`: 已用额度（Long，分）— 用户设置初始值，后续自动更新
- **可用额度 = total_limit - used_amount**（实时计算，不存储）
- `bill_day`: 账单日（1-31）
- `repayment_day`: 还款日（1-31）

**重要：信用账户是做减法的逻辑**
- 消费（支出）→ used_amount 增加
- 还款（从资金账户转账到信用账户）→ used_amount 减少
- 页面显示：可用额度 / 总额度

### 4.4 投资账户

| 子类型 | 说明 |
|--------|------|
| 养老账户 | 个人养老金账户 |
| 定存 | 银行定期存款 |
| 股票账户 | 证券账户 |
| 期货账户 | 期货交易账户 |

字段定义：
- `name`: 账户名称
- `balance`: 当前市值（Long，分）— 手动更新
- 投资账户不参与日常收支记录，仅手动调整市值

### 4.5 贷款账户

| 子类型 | 说明 |
|--------|------|
| 房贷 | 住房贷款 |
| 车贷 | 汽车贷款 |
| 消费贷 | 消费信贷 |

字段定义：
- `name`: 账户名称
- `total_loan`: 贷款总额（Long，分）
- `already_paid`: 已还金额（Long，分）
- **剩余贷款 = total_loan - already_paid**
- `monthly_payment`: 月供金额（Long，分）
- `repayment_day`: 每月还款日（1-31）

还款规则：
- 还款（从资金账户转账到贷款账户）→ already_paid 增加，资金账户 balance 减少

## 6. 设置模块 (P1)

### 6.1 分类管理

**默认支出一级分类**：餐饮、生活、娱乐、交通、教育

**默认支出二级分类**：
- 餐饮 → 早餐、午餐、晚餐、零食、饮料
- 生活 → 日用品、水电燃气、房租、物业
- 娱乐 → 电影、游戏、旅行
- 交通 → 公交地铁、打车、加油
- 教育 → 书籍、课程、培训

**默认收入一级分类**：工资、兼职、理财收益、其他

操作：
- 新增/编辑/删除一级分类和二级分类
- 拖拽排序
- 自定义图标
- 删除规则：有关联交易记录的分类仅隐藏（is_deleted=true），不物理删除
- 系统默认分类可隐藏但不可物理删除

### 6.2 预算设置

两种模式**互斥**，同一时间只能启用一种：

| 模式 | 说明 |
|------|------|
| TOTAL（总额模式） | 设置一个月度总预算金额 |
| PER_CATEGORY（分类模式） | 为每个一级支出分类分别设置月预算 |

- 切换模式时弹出确认对话框，提示将清除当前模式的预算数据
- 预算按自然月计算（可配置起始日）
- 首页预算卡片根据当前模式显示对应数据

### 6.3 通用设置

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 主题 | 跟随系统 | 浅色/深色/跟随系统 |
| 货币符号 | ¥ | 可选 ¥ $ € 等 |
| 每月起始日 | 1 | 1-28，影响月度统计的起止范围 |
| 检查更新 | 开启 | 自动检查 GitHub 新版本 |

- 导入完成后显示结果摘要

## 8. 定期扣款模块 (P1)

### 8.1 定期模板

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | 是 | 如"招行信用卡还款"、"房贷月供" |
| 金额 | 是 | 每次执行金额（分） |
| 频率 | 是 | 每月 / 每周 / 自定义天数 |
| 执行日 | 是 | 每月几号 / 每周几 |
| 源账户 | 是 | 扣款方（如工资卡） |
| 目标账户 | 是 | 收款方（如信用卡、房贷账户） |
| 是否启用 | 是 | 默认启用 |

适用场景：
- 信用卡自动还款：资金账户 → 信用账户（used_amount 减少）
- 房贷/车贷月供：资金账户 → 贷款账户（already_paid 增加）
- 定投扣款：资金账户 → 投资账户（balance 增加）

### 8.2 执行逻辑

1. WorkManager 每日检查 `next_due_date <= 今天` 且 `is_enabled = true` 的模板
2. 对每个到期模板：
   - 创建一条 TRANSFER 类型的 Transaction
   - 更新源账户和目标账户的余额/额度
   - 计算并更新 `next_due_date`
3. 所有操作在 Room @Transaction 中执行，保证原子性
4. 执行成功发送通知："已自动执行 [模板名称] ¥xxx"
5. 执行失败（如余额不足）发送警告通知

### 8.3 管理界面

- 列表展示所有定期任务，显示名称、金额、频率、下次执行日
- 开关控制启用/暂停
- 点击编辑，长按删除
- 底部"新增定期任务"按钮

## 9. 热更新模块 (P2)

### 9.1 版本检查

- 调用 GitHub Releases API：`GET https://api.github.com/repos/{owner}/{repo}/releases/latest`
- 比较返回的 `tag_name` 与本地 `BuildConfig.VERSION_NAME`
- WorkManager 每日后台检查一次（可在设置中关闭）

### 9.2 更新流程

1. 发现新版本 → 弹出 Material AlertDialog
2. 显示：新版本号 + 更新日志（从 release body 解析）
3. 用户点击"立即更新" → DownloadManager 下载 APK
4. 下载完成 → 引导安装（需 REQUEST_INSTALL_PACKAGES 权限）
5. 用户点击"稍后" → 关闭弹窗，下次启动再提醒

### 9.3 强制更新

- Release 中可设置 `min_version` 标签
- 低于 min_version 的版本弹出不可关闭的更新弹窗

## 10. 非功能性需求

| 维度 | 要求 |
|------|------|
| 性能 | 列表滚动 60fps，数据库查询 <100ms |
| 存储 | 数据库 <50MB（正常使用 3 年） |
| 兼容性 | Android 8.0 (API 26) 及以上 |
| 无障碍 | 支持 TalkBack，触摸目标 ≥48dp |
| 隐私 | 纯本地存储，零网络权限（除热更新） |
| 安全 | 数据库存储在 app 私有目录，导出可加密 |
